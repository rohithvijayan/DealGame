  1. EXECUTIVE SUMMARY  
1.1 Product Vision
The Congress BJP Deal is a mobile-first political trivia game that exposes India's most significant party defections through an addictive, gamified experience. Players act as intelligence analysts tasked with identifying Congress leaders who crossed over to the BJP between 2014 and 2026.
"Congress Today. BJP Tomorrow." — The tagline encapsulates the game's central thesis: that political allegiance in India is transactional, and that voters deserve to know who made the deal.
1.2 Opportunity
India's political defection phenomenon — colloquially called 'aaya ram gaya ram' — has accelerated dramatically since 2014. Yet public awareness of individual defectors remains fragmented across news archives. This game consolidates 60+ documented defections into an engaging, shareable mobile experience that is equal parts entertainment and civic education.

2.1 Core Game Loop
Each game session consists of 10 rounds. In each round the player is shown a classified dossier containing contextual clues about a defector — their former Congress role, home state, year of crossing over, and an intel clue — but no name or photo. The player must identify the person by typing their name.

4.5 Name Matching Algorithm
Guesses are validated server-side via /api/guess to prevent client-side answer exposure. The matching algorithm uses the following priority chain:
Exact full-name match (case-insensitive, trimmed)
Alias match — check all entries in the aliases[] array
Substring match — if guess length ≥ 4 and is fully contained in canonical name
Token match — if any significant name token (>3 chars) matches a token in the guess
Levenshtein distance ≤ 2 for inputs ≥ 6 characters (handles typos)
Security note: The canonical name, aliases, and hints array are NEVER sent to the client before a correct guess. Only the clue, position, state, and year fields are exposed. Validation happens exclusively on the server.
  5. ANIMATION & MOTION SPEC  
5.1 Core Principles
All transitions under 400ms — this is a fast-paced game, not a portfolio site
Spring physics preferred over linear easing — use Framer Motion spring with stiffness 300, damping 20
Every animation must have a clear game-feel purpose: surprise, reward, punishment, or pacing
Reduce motion: respect prefers-reduced-motion — replace all animations with instant transitions

  7. MOBILE UX REQUIREMENTS  
7.1 Layout Constraints
Single column layout throughout — no horizontal scroll anywhere
Dossier card: 55–60% of viewport height (dvh units)
Input + CTA zone: always in bottom 40% of screen, always visible without scroll
All tap targets minimum 48×48px, preferred 56px height for primary CTAs
Safe area padding: env(safe-area-inset-*) on all screen edges — notch + home bar
Keyboard-aware layout: use dvh units, card compresses gracefully when keyboard opens
7.2 Touch Interactions
Swipe left on dossier card = SKIP (with confirmation haptic)
Long press on clue text = request first hint (discovery mechanic)
Double-tap on Deal Done overlay = advance to next round
Haptic feedback (navigator.vibrate where supported): correct answer (double pulse), wrong (single sharp), streak (triple)
7.3 PWA Requirements
manifest.json: name, short_name, theme_color (#FF6B00), background_color (#1C1C1C), display: standalone
Offline support: Service Worker caches defector data + assets — game fully playable offline
Add to Home Screen prompt triggered after 3rd session
Splash screen matches game aesthetic — saffron logo on near-black background
7.4 Performance Targets
LCP < 1.5s on 4G (Vercel Edge + CDN)
CLS = 0 — no layout shift on card entry (reserve card height before load)
FID < 50ms — input response must feel instant
Total JS bundle < 180KB gzipped (lazy-load Framer Motion + game screen)
Defector data: first 10 entries pre-fetched at session start, remainder on demand

  10. OPEN QUESTIONS  
Pending Decisions
Photo sourcing strategy — Wikimedia Commons (free) vs licensed press photos? Wikimedia preferred for MVP cost and legal clarity.
Anonymous vs required login for leaderboard — recommend anonymous sessions with optional name claim for Phase 2.
Hindi localization in Phase 2 or Phase 3? Large TAM upside but significant content effort.
Should the game also cover BJP → Congress defections for balance? Smaller dataset but reduces legal/perception risk.
App Store distribution (React Native wrapper) — worth the overhead vs PWA-first approach?
Monetization: No ads in MVP. Phase 3 options: cosmetic dossier themes, premium daily unlock early access. No paywall on core game.
